# AI 施工管理系统 - 实现方案（对照需求）

## 1. 技术方案简述

### 1.1 图像与多模态能力（全链路谷歌多模态大模型）
- **统一基座**：AI 图像识别/视频理解/图文比对/报修多模态分类等模块，**一律通过 谷歌多模态大模型** 实现，例如 **Gemini 1.5/2 Pro** 系列。
- **接入方式**：
  - 海外或合规环境：**Google AI Studio / Vertex AI**（`generateContent` + 多模态 parts：图片/短视频片段）。
  - 需落地合规与审计：在业务侧对请求与回包脱敏、日志留存；敏感图档可增加水印与分片上传。
- **不采用**：YOLO 等独立小模型作主线（避免与“全多模态”要求冲突）；若需几何级精度可二期叠加专用检测，本期以 Gemini 结果 + **规则引擎** 为主。
- **视频处理**：按关键帧/片段抽取后送入多模态输入；大文件走异步任务队列。
- **变化检测**：同点位多期影像 + 模型生成对比说明与差异标签，阈值由规则引擎与人工复核配合。

### 1.2 系统架构
- 前端：
  - **管理后台**：Vue 3 + TypeScript + Ant Design Vue。
  - **项目专属 H5/网站**：Vue/React 单页，按项目子域或 path 部署（如 `project-xxx.yourdomain.com`）。
  - **专属 APK**：RN / Flutter / Uni-App 壳 + **热配置**（项目 ID、鉴权、皮肤、包名、应用名）；**CI 流水线** 触发打包签名输出 APK（可对接蒲公英/自建下载页）。
- 后端：
  - 单体或轻量微服务：Node（Nest）或 Java（Spring Boot），与 **1.5 个月** 周期匹配、降低联调面。
  - 核心服务：用户与 RBAC、项目/工地、任务工作流、文件、**多模态 AI 代理服务**、**消息与聊天（WebSocket）**、**实时进度聚合**、报表。
- AI 服务层：
  - **Google Gemini 客户端封装**（重试、限流、多 Key 轮询、流式/非流式、结构化输出 JSON Schema）。
  - 业务 **规则引擎**（工序/验收/风险评分），与模型输出解耦，便于调参与合规。
- 部署：云服务器 + OSS + HTTPS；实时能力依赖 **WSS/Redis PubSub** 或托管实时通道（按量选型）。

### 1.3 数据存储方案
- 结构化数据：MySQL 或 PostgreSQL
- 媒体：对象存储（OSS/MinIO/S3 兼容）
- 缓存/会话：Redis
- 队列：RabbitMQ / 云 SQS 类，承载异步 Gemini 调用、日报生成
- 聊天：消息表 + 已读 + 轻量未读；在线列表可用 Redis
- 日志与审计：按角色脱敏的访问与操作日志

### 1.4 工作流实现方式
- 工作流：优先 **状态机**（可嵌入 Flowable 若后期扩展）或自研表驱动流程，匹配短周期。
- 流程状态：待派发 → 进行中 → 待验收 → 整改中 → 已完成
- 事件触发：上传影像经队列调用 Gemini → 规则引擎 → 自动建整改/通知；超时催办

### 1.5 专属进度 APK 与网站（自动生成）
- **管理端一键**：新建/完工项目时生成 `projectId`、下载用 **短链**、可选 **白标**（名称、色板、Icon）。
- **APK 流水线**：`main` 分支 + `config.json` 注入 + Android 自动签名 + 制品归档（Oss 可下载安装）。
- **网站**：同后端 API、同鉴权，支持客户 **实时** 看进度、影像、与 **工程聊天** 同步出现。

## 2. 对照需求的实现映射
- AI 图像识别：**Google Gemini 多模态** + 规则引擎 + 异步任务
- 工作流：流程状态 + 消息 + RBAC
- 专属终端：**APK 生成流水线 + 专属站点 + 实时 + 项目内 IM**
- TOC/材料/远程观测/售后：同一后端数据模型，客户侧主要走 **网站与 APK**；售后多模态分类同走 Gemini
