项目背景与需求范围

一、项目背景

我司为建筑装修设计公司，现需开发/部署 AI 施工实时聊天监控系统，用于提升工地管理效率、减少沟通成本、实现现场进度可视化与多方即时协同。

二、核心功能模块（按优先级排序）

按建设目标整理的需求要点说明。

1. AI 图像识别 输入：设计图、施工图、效果图、现场照片/视频。功能：比对现场与图纸一致性；识别施工错误；多阶段图像存档与变化检测。

2. 工作流协同 角色：工人、客户、施工经理、总承包方。功能：任务派发与接收、现场问题闭环、各方实时查看进度与异常提醒。

3. 进度与材料（TOC） 材料进场/使用/剩余可视化；基于 TOC 识别瓶颈工序；延期风险预警与调度建议。

4. 客户远程观测 重点交付 一客一专属 App + H5/网站/小程序：实时形象进度、现场/节点、评论/验收/提问；即时工程内沟通。可选 AI 日/周摘要。

5. 售后管理 售后档案、报修（多模态）、AI 分类派单、维修跟踪与满意度回访。

三、技术要求（建议方案包含）

  • 支持云端部署或本地+云端混合
  • 移动端（微信小程序或 App）+ 网页管理后台
  • 图像识别可私有化或可商用 API
  • 数据权限分级（不同角色见不同内容）

  • 总报价（可分开首期+后期扩展）
  • 维护/迭代费用说明
  • 是否提供试用/POC 环境

技术实现方案

1.1 多模态（全链路 谷歌多模态模型）

  • 统一基座：识别/视频/比对/报修多模态分类经 谷歌多模态模型，经官方多模态云端服务接入，支持图文/短视频等复合输入
  • 合规与审计：脱敏、日志、水印与分片上传；几何精度专检可二期叠加
  • 视频：关键帧/片段、异步队列；变化检测：多期对比 + 规则 + 人工复核

1.2 系统架构

  • 前端：管理后台 Vue3+TS+Ant Design；专属 H5/网站；微信小程序；专属 APK（RN/Flutter/Uni-App+热配置+CI）
  • 后端：Nest 或 Spring Boot；用户 RBAC、项目/工地、工作流、文件、多模态代理、WebSocket 聊天与实时、报表
  • AI：谷歌多模态模型 服务封装（重试、限流、多路轮换、流式/结构化 JSON）；规则引擎 与模型解耦
  • 部署：云服务器 + OSS + HTTPS；实时 WSS/Redis PubSub 等

1.3 存储

MySQL/PostgreSQL、对象存储、Redis、RabbitMQ/SQS、消息表+已读、按角色脱敏的审计日志。

1.4 工作流

待派发 → 进行中 → 待验收 → 整改中 → 已完成；上传经队列调 谷歌多模态模型 → 规则 → 整改/通知/催办。

1.5 专属进度（网站/APK/小程序 同源）

  • 管理端：projectId、短链、白标、APK 流水线、下载页、小程序发版/体验码
  • 网站与多客户端同 API、同鉴权，实时+聊天同屏
