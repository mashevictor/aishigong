索引 AI 能力落点登记表（演示仓库）

汇总本站页面 · HTTP 接口 · 数据 JSON · 构建脚本中与 AI / 算法 / 大模型代理相关之处；紫色角标 AI 在各业务页含义与此一致。

前端页面 后端接口 数据与离线包 npm 脚本 演示沙盘 矩阵·闸口专页

说明 「AI」包含：已接通的模型代理（DeepSeek / Wavespeed）、前端展示的 AI 叙事与举证字段、以及预留的多模态 / 图纸比对占位；不构成法律承诺。

一、前端静态页（标注 AI 关联）

下列路径均由 Node 静态托管或 CDN 直接提供；「AI 关联」简述用户可见或与 AI 数据同源的部分。

路径 AI 关联
/ai-showcase.html 全页：五维矩阵、验收闸口、国标/DG 叙事、DeepSeek 刷新说明；数据 GET /api/meta/ai-showcase 或 fallback。
/contract-ai.html 合同/采购/工时侧 AI 摘要、价差与供应商「AI 结论」演示；GET /api/meta/procurement-ai。
/multimodal.html API 展示 ai_logs 调用留痕（DeepSeek/Wavespeed 等）；管理员只读。
/index.html 方案正文含多模态与谷歌模型叙事；工作台 /api/media/dashboard 含 AI 留痕摘要；视图「AI 文生文 / 文生图」走 POST /api/ai/chat、POST /api/ai/image。
/demo.html 问题全景矩阵 AI 列；工序与矩阵页能力映射块；typical_issues 含 AI 话术。
/scenario-120.html room_cases 条目的 AI 价值字段；主线讲解衔接验收闸口。
/manager.html 快捷入口指向矩阵页、合同 AI；看板本身为业务汇总，常与 AI 闸口话术串联讲解。
/portal.html 模块网格含「AI + 建造矩阵」「合同·采购 AI」「多模态」等入口。
/client.html 效果图/实拍墙与 milestones — 竣工验收 AI 叙事中的业主可视证据链（演示数据）。
/materials.html 材料 TOC / 影像 — 可与合同 AI「清单 ↔ 到货」对照叙事联动。
/tickets.html 质保工单 — 矩阵页「工单语义分级 / 质保闸口」占位叙事。
/worker.html 任务影像上传 — 可作为多模态分析输入源的演示链路一环。
/admin.html 运营统计；管理员可查 GET /api/admin/ai-logs（本页 UI 未展开时可从主站工作台预览摘要）。
/handover.html 移交清单表列出多模态页与 AI 代理路由说明。
本页 /ai-capability-map.html 全站 AI 落点索引（纯静态登记表）。

二、后端 HTTP（直接与 AI / AI 叙事数据相关）

路由 说明
POST /api/ai/chat 需登录；转发 DeepSeek Chat Completions，写入 ai_logs。
POST /api/ai/image 需登录；Wavespeed 文生图代理，写入 ai_logs。
GET /api/admin/ai-logs 管理员；分页读取调用留痕。
GET /api/meta/ai-showcase AI + 建造矩阵与闸口 JSON（server/ai-construction-showcase.json）。
GET /api/meta/procurement-ai 合同/采购演示 JSON（server/procurement-ai-demo.json）。
GET /api/meta/scenario 含 typical_issues、room_cases、demo_issues_matrix 等 AI 举证列 / ai_value 字段。
GET /api/media/dashboard 工作台一览；管理员响应中带 AI 调用摘要片段。

其余业务接口（任务、工单、材料）可为 AI 比对提供数据来源，但不直接调用外部模型。

三、数据文件与离线包（内容含 AI 字段或叙事）

路径（仓库内） 说明
server/ai-construction-showcase.json 矩阵页主数据；闸口、国标对照、DeepSeek 说明。
public/assets/ai-showcase-fallback.json 接口不可用时降级数据包。
server/procurement-ai-demo.json 合同 AI 页数据源。
server/demo-issues-matrix.json 沙盘「典型问题」矩阵；含 AI 列文案。
public/assets/demo-issues-fallback.json 矩阵离线兜底。
server/scenario-120-room-cases.json 按房间争议与 ai_value。
server/media.js（内嵌 typical_issues） 若干条明确写「隐蔽验收 AI」「竣工 AI」等演示提要。
server/scenario-image-specs.json Wavespeed 出图规格（脚本生成沙盘配图）。

四、构建脚本（调用大模型或文生图 API）

在 server/ 目录执行；需对应环境变量中的 API Key。

命令 脚本 说明
npm run showcase:deepseek scripts/enrich-ai-showcase-deepseek.mjs DeepSeek 刷新矩阵 JSON 段落。
npm run case:rooms-deepseek scripts/enrich-room-cases-deepseek.mjs DeepSeek 补充房间争议案例。
npm run scenario:images / scenario:images:cn-four scripts/generate-scenario-images.mjs Wavespeed 批量出图；可选 DeepSeek 优化 prompt。
统一门户 · 演示沙盘 · GET /api/meta/delivery 交付物摘要（亦含 AI 相关路由提要）
