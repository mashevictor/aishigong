# AI 施工监控系统 · 角色可见范围与页面功能梳理

> 本文描述**当前仓库已实现的**前后端能力；微信小程序、定制 APK、完整 OSS/网关等为**独立交付物**，不在此仓库内完整实现，文中单独标明。

---

## 一、总体架构（「有多少系统」）

| 层级 | 名称 | 说明 |
|------|------|------|
| **运行时** | 单一 Node 服务 | `server/index.js`，默认端口见根目录 `.env` 的 `PORT`（如 3780） |
| **子系统 / 入口（前端形态）** | **5 个独立 HTML 站点入口** | 同一域名不同路径，共享浏览器本地令牌 `localStorage.aishigong_token` |
| **独立产品（未在本仓库）** | 微信小程序、一客一 APK | 需单独工程；接口设计上与本站 REST API 同源即可 |

**结论（便于对外汇报）：**

- **1 套后端服务** + **5 个前端页面（站点入口）** = 当前演示型「全栈站点」。
- **业务子域**可划分为：**门户与协同主站**、**管理后台**、**客户观测 H5**、**多模态留痕**、**移交说明**（共 5 类用户可见入口）。

---

## 二、页面清单（「有多少页面」）

| # | 路径 | 名称 | 主要对象 |
|---|------|------|----------|
| 1 | `/index.html` | **主站**：方案正文 + 施工协同工作台 + AI 文生文/图 | 全员（按登录角色限制数据） |
| 2 | `/admin.html` | **Web 管理后台** | 仅 **管理员** |
| 3 | `/client.html` | **客户远程观测（H5 视图）** | 能登录且被授权到项目的用户（面向业主演示用 **客户** 账号） |
| 4 | `/multimodal.html` | **多模态分析结论区**（AI 调用 `ai_logs` 列表） | 仅 **管理员** |

**附加静态资源（非独立「系统」）：**

- `/assets/demo/concrete.svg`、`/assets/demo/helmet.svg`：外链图片失败时的本地占位。
- 仓库内另有历史方案 PDF/HTML（如 `方案汇总0424备份.html`），**未全部接入**主导航，可按需链出。

**机器可读元数据：** `GET /api/meta/delivery` 返回页面与路由组的 JSON 摘要。

## 三、角色定义（RBAC + 项目授权）

全局角色（存储于 `users.role`，枚举）：

| 角色 | 代码值（中文） |
|------|----------------|
| 管理员 | `管理员` |
| 施工经理 | `施工经理` |
| 工人 | `工人` |
| 客户 | `客户` |
| 总包 | `总包` |
| 售后 | `售后` |

**数据可见性规则（核心）：**

1. **管理员**：可见**所有项目**及下属任务、消息（含管理接口）。
2. **非管理员**：仅可见 `project_members` 表中**本人被授权**的 `project_id` 下的项目、任务、消息、客户视图聚合等。
3. **登录凭证**：JWT（根目录 `.env` 必须配置 `JWT_SECRET`，长度 ≥ 16）。

演示账号（密码均为 **`demo123`**）：`admin`、`jingli`、`gongren`、`kehu`、`zongbao`、`shouhou`。

## 四、分角色 · 能看见哪些前端与功能

### 1. 管理员（`admin`）

| 入口 | 能看见 / 能做什么 |
|------|-------------------|
| **主站 `index.html`** | 方案汇总全文；**工作台**（登录后）：项目卡片+封面、模拟图库、`/api/media/dashboard` 中的 **AI 留痕摘要**、项目表格、任务创建与状态流转、工程聊天、刷新消息轮询；**AI 文生文/图** |
| **`admin.html`** | 运营统计、用户列表、项目新建（含封面 URL）、进度微调、项目成员增删、退出 |
| **`client.html`** | 与其他角色相同逻辑：仅**已授权项目**的客户视图（可用于验收演示） |
| **`multimodal.html`** | `ai_logs` 最近记录列表 |
| **`handover.html`** | 交付映射说明 |

**不可越权：** 无单独「超级页面」；所有写操作仍受后端校验。

## 五、按「页面」梳理功能点（能力矩阵）

| 顶栏链接 | 跳转 admin / client / multimodal / handover | ✅ | ✅ | ✅ |

|------|------|

## 六、后端接口分组（与「能看见什么」对应）

| 分组 | 路径模式 | 典型角色 |
|------|----------|----------|
| 健康检查 | `GET /api/health` | 公开 |
| 登录 | `POST /api/auth/login` | 公开 |
| 当前用户 | `GET /api/me` | 已登录 |
| 业务协同 | `GET/POST /api/projects`（仅 GET 已实现列表）、`GET/POST/PATCH /api/tasks`、`GET/POST /api/messages` | 已登录 + 项目授权 |
| 客户聚合 | `GET /api/client/overview` | 已登录 + 项目授权 |
| 媒体一览 | `GET /api/media/dashboard` | 已登录；AI 摘要仅管理员 |
| 管理 | `/api/admin/*` | **仅管理员** |
| 工作流元数据 | `GET /api/meta/workflow` | 公开 |
| 交付元数据 | `GET /api/meta/delivery` | 公开 |
| SSE 占位 | `GET /api/stream/events` | 已登录（心跳，非完整 IM 推送） |
| AI 代理 | `POST /api/ai/chat`、`POST /api/ai/image` | 公开（密钥在服务端） |

## 七、已知局限（与「太简单」的对照说明）

1. **实时**：仅为 SSE 心跳示例，**不是**完整 WebSocket 多端推送。
2. **小程序 / APK**：未在本仓库实现界面与发版流程。
3. **审计**：任务状态变更有日志需求时，需单独 `audit_log` 表（当前节点留痕由任务列表推导）。
4. **上传**：现场照片**未**做专用上传接口；封面 / 图库当前为 **URL + 演示外链 + 本地 SVG**。
5. **权限细化**：任务「谁能创建 / 谁能改状态」尚未细分到字段级策略，统一为「项目成员即可」为主。
