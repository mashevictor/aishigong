# 生产环境部署说明

本文档面向「可直接上线」的配置与安全默认值；行为以代码为准（`server/index.js`、`server/db.js`）。

## 1. 运行环境

- **Node.js**：建议 **18 LTS** 或 **20 LTS**。
- **MySQL**：**8.x**（utf8mb4）。库表可由 Node 首次启动自动 `CREATE`，也可用 `server/sql/init_mysql.sql` 预建库。
- **反向代理**：生产默认 Node 监听 **`127.0.0.1:PORT`**，对外由 **Nginx（HTTPS）** 反代，见 `deploy/nginx.aishigong.example.conf`。

## 2. 目录与依赖

典型服务器路径示例：`/var/www/aishigong`。

```bash
cd /var/www/aishigong/server
npm ci --omit=dev
```

## 3. 环境变量（`.env`）

将仓库根目录 **`.env.example`** 复制为 **`.env`**（与 `server/` 同级）。至少配置：

| 变量 | 说明 |
|------|------|
| `NODE_ENV` | 设为 **`production`** |
| `JWT_SECRET` | **≥16 字符**随机串 |
| `MYSQL_*` | 生产 **必填**；缺失将导致进程 **退出**（禁止静默使用内存库） |
| `TRUST_PROXY` | **仅**在 Nginx/Caddy 等 **反代后面** 设为 **`true`**；若浏览器 **直连 `http://IP:端口`**，必须 **关闭或未设置**，否则会误判客户端 IP，`express-rate-limit` 可能报错导致 **`GET /` 返回 500** |

### 3.1 首次空库：管理员从哪里来？

生产环境默认 **`SEED_DEMO_DATA` 为关闭**（等价于 `false`）：**不会**写入 `demo123` 等演示账号与演示项目。

任选其一：

- **推荐**：配置 **`ADMIN_BOOTSTRAP_USERNAME`**、**`ADMIN_BOOTSTRAP_PASSWORD`**（生产下密码至少 **12** 位）、可选 **`ADMIN_BOOTSTRAP_DISPLAY_NAME`**。仅在 **`users` 表为空** 时插入一名 **管理员**。登录验证通过后，**请从环境中删除**上述明文密码变量。
- **演示/内网**：临时设置 **`SEED_DEMO_DATA=true`**，重启服务写入演示数据（务必随后改密或关闭该变量）。

### 3.2 静态文件范围

生产默认 **仅挂载 `public/`**。仓库根目录下的方案 HTML、图片等 **默认不暴露**；若确需托管，设置 **`SERVE_REPO_ROOT=true`**（慎用）。

### 3.3 跨域

若浏览器访问的前端域名与接口域名不一致，设置 **`CORS_ORIGIN`**（逗号分隔多个源）。生产环境下未配置时 CORS 为关闭模式，仅适合 **同域** 部署。

## 4. 进程管理

可使用 **PM2**（示例：`deploy/pm2.ecosystem.example.cjs`）或 **systemd**。务必在进程环境中传入 **`NODE_ENV=production`**。

PM2 示例：

```bash
cd /var/www/aishigong/server
NODE_ENV=production pm2 start index.js --name aishigong
pm2 save
```

（可将环境变量写入 pm2 ecosystem 的 `env_production`。）

## 5. 健康检查

| 路径 | 用途 |
|------|------|
| `GET /api/health` | 存活检测（不查库） |
| `GET /api/health/ready` | **就绪检测**（MySQL `SELECT 1`）；生产若以内存 mock 运行会 **503** |

负载均衡 / K8s 建议：**就绪探针** 使用 **`/api/health/ready`**。

## 6. 安全特性摘要

- **Helmet** 安全响应头（已关闭 CSP 以兼容现有内联脚本页面）。
- **全局限流**（可 `RATE_LIMIT_*` 调整）；**登录** 单独限流（`LOGIN_RATE_LIMIT_*`）。
- **AI 接口**需 JWT，避免匿名滥用密钥。
- 生产 **禁止** 未配置 MySQL 时回落内存库；MySQL 连接失败 **直接退出**。

## 7. 备份与升级

- 定期备份 **MySQL** 与 **`.env`**（勿入库）。
- 升级：`git pull` → `npm ci --omit=dev` → 重启进程；表结构由启动时的 `ensureMysqlSchema` 增量补齐。

## 8. 防火墙与监听

默认 **`BIND_HOST=127.0.0.1`**，仅本机访问；由 Nginx 对外。若必须 Node 直挂公网，改为 **`BIND_HOST=0.0.0.0`** 并配合防火墙仅放行 80/443。
