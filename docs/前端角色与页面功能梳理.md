# 前端角色可见性与页面功能梳理

> **适用范围**：本仓库当前实现（Node + Express + 静态 HTML + MySQL/内存模拟）。  
> **权限模型**：JWT 登录后，接口根据 **全局角色**（`users.role`）与 **项目成员表**（`project_members`）决定可见范围。**管理员**默认可见全部项目；其余角色仅可见已授权项目的数据。

---

## 一、有多少「系统」？

本文档将「系统」分为三类，避免与「页面」混淆：

| 序号 | 系统名称 | 形态 | 说明 |
|------|----------|------|------|
| **S1** | **主 Web 应用（本站）** | 浏览器访问同一域名下的多个 HTML + 共用 `/api/*` | 当前交付主体：`public/` 下页面 + `server/` 后端。 |
| **S2** | **微信小程序** | 独立仓库 / 微信开发者工具 | **未在本仓库实现**，仅方案与 `handover.html` 中说明；上线后应对接同一套 API 与鉴权。 |
| **S3** | **一客一专属 APK** | 独立 App 工程（RN/Flutter/Uni-App 等） | **未在本仓库实现**；业务数据仍来自同一后端 API。 |

**小结**：代码层面完整落地的 **Web 系统 = 1 套（S1）**；规划中的 **多端扩展 = 2 类（S2、S3）**，需单独立项开发与发版。

---

## 二、有多少「页面」（本仓库内）？

以下为 **随 Node 静态托管、可直接 URL 访问** 的前端页面统计：

| 序号 | 路径 | 名称 | 是否需要登录 |
|------|------|------|----------------|
| **P1** | `/portal.html` | 统一门户（按角色模块导航） | **必须登录** |
| **P2** | `/index.html` | 主站（方案汇总 + 工作台 + AI） | 方案区可无登录；**工作台与 AI 接口需登录** |
| **P3** | `/manager.html` | 施工经理数据看板 | **必须登录** |
| **P4** | `/worker.html` | 工人极简（任务 + 影像只读） | **必须登录** |
| **P5** | `/materials.html` | 材料 TOC + 现场影像归档 | **必须登录** |
| **P6** | `/tickets.html` | 售后质保工单 | **必须登录** |
| **P7** | `/admin.html` | Web 管理后台 | **必须登录且角色 = 管理员** |
| **P8** | `/client.html` | 客户远程观测（H5 视图） | **必须登录**（按项目授权过滤数据） |
| **P9** | `/multimodal.html` | 多模态 / AI 调用留痕 | **必须登录且角色 = 管理员**（列表接口受限） |
| **P10** | `/handover.html` | 交付物与映射说明 | **无需登录**（静态说明） |

**合计：10 个标准页面（P1～P10）。**

另：仓库根目录若有其它 `.html`（如历史方案备份），在开启 `express.static(项目根)` 时可能被访问到，**不作为产品菜单内的标准页面**；正式对外建议只暴露 `public/` 与上述路由。

## 三、全局角色说明（RBAC）

数据库枚举角色（与登录 JWT 一致）：

| 全局角色 | 典型用途 |
|----------|----------|
| **管理员** | 跨项目运营、用户列表、项目 CRUD、成员授权、AI 日志全库查看 |
| **施工经理** | 所辖项目的任务、沟通、进度协同 |
| **工人** | 现场任务、留言（范围限于授权项目） |
| **客户** | 远程看进度、节点与留言（偏 `client.html` 场景） |
| **总包** | 多项目视角下的协同（演示数据中示例为项目 2） |
| **售后** | 工单类扩展占位（演示数据中参与部分项目） |

**数据隔离规则（概要）**：

- `/api/projects`、`/api/tasks`、`/api/messages`、`/api/client/overview`、`/api/media/dashboard`、`/api/materials`、`/api/site-photos`、`/api/tickets`、`/api/dashboard/summary`、`/api/portal/modules`：**仅返回或写入当前用户有权访问的项目及其下属数据**。
- `/api/admin/*`：**仅「管理员」**。
- `/api/ai/chat`、`/api/ai/image`：**需登录**（JWT）；密钥仍只在服务端，前端仅携带令牌调用代理。

## 五、按「页面」梳理：各页功能与谁该用

### P1 — `portal.html`（统一门户）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 模块卡片 | 调用 `/api/portal/modules`，按全局角色过滤可点入口 | **已登录** |

---

### P2 — `index.html`（主站）

| 模块 | 功能 | 谁「能看见/能用」 |
|------|------|-------------------|
| **顶栏导航** | 切换：方案汇总 / 工作台 / AI；外链到门户、经理看板、材料、工单、工人视图及管理端等 | 所有人 |
| **方案汇总** | 完整商务与技术方案长文 | 所有人（只读） |
| **工作台** | 登录、我的项目（表+卡片+封面）、模拟图库条、**管理员**可见 AI 留痕摘要、任务增改、工程聊天、`/api/media/dashboard` 等 | **必须登录**；数据按项目授权过滤 |
| **AI 文生文 / 文生图** | DeepSeek / Wavespeed | **必须登录**（JWT）；未登录时前端提示先到工作台登录 |

**注意**：非管理员在工作台 **看不到** 管理类数据（如全站用户列表、全站项目配置），但可看到自己授权范围内的项目与任务。

---

### P3 — `manager.html`（施工经理看板）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 汇总 | `/api/dashboard/summary`：任务按状态计数、材料条数、影像条数、未关闭工单数 | **已登录**（数据按项目授权） |

---

### P4 — `worker.html`（工人极简）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 任务 | `/api/tasks` 列表与 `PATCH` 更新状态 | **已登录**；门户中入口限工人/经理/管理员，URL 直连仍以接口权限为准 |
| 影像 | `/api/site-photos` 只读展示 | 同上 |

---

### P5 — `materials.html`（材料与影像）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 材料 TOC | `GET/POST /api/materials` | **已登录** |
| 现场影像 | `GET/POST /api/site-photos`（演示用 URL 建档） | **已登录** |

---

### P6 — `tickets.html`（售后工单）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 工单 | `GET/POST /api/tickets`、`PATCH /api/tickets/:id` | **已登录** |

---

### P7 — `admin.html`（管理后台）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 运营概览统计 | 项目/用户/任务/消息/AI 日志条数 | **仅管理员** |
| 用户与 RBAC 列表 | 全站用户与角色 | **仅管理员** |
| 项目配置 | 新建项目、调整进度、**封面图 URL** | **仅管理员** |
| 项目成员 | 增删 `project_members` | **仅管理员** |

**非管理员**：即使手动打开 URL，登录后会被前端提示并拒绝；接口 `/api/admin/*` 也会 **403**。

---

### P8 — `client.html`（客户 H5 视图）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 项目选择 | 下拉仅含已授权项目 | 所有**已登录**且被授权者（不仅限客户） |
| 封面与进度 | 项目 `cover_image_url`、进度条 | 同上；无项目则列表为空 |
| 现场影像（演示） | 与概览接口返回的 `gallery` 展示 | 同上 |
| 节点与留痕 | 由任务状态推导的轻量里程碑 | 同上 |
| 工程留言 | 拉取/发送 `messages` | 同上，且仅能操作有权的 `project_id` |

**产品定位**：给 **业主/客户** 用；**施工经理、工人** 若被授权到同一项目，技术上也可见该页（便于联调），生产可再在前端按角色隐藏入口。

---

### P9 — `multimodal.html`（多模态结论区）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| AI 调用留痕列表 | 读 `ai_logs` | **仅管理员**（接口 `/api/admin/ai-logs`） |

**非管理员**：无法拉取完整列表；日常 AI 使用仍可在 P2（主站）的 AI 区进行，但留痕全表仅管理端可见。

---

### P10 — `handover.html`（移交说明）

| 功能 | 说明 | 可见角色 |
|------|------|----------|
| 交付物与页面映射、系统边界 | 静态说明 + 链到各页 | **所有人**（只读） |

## 六、按「角色」梳理：登录后各角色「建议看哪些页、能操作什么」

下表描述 **在当前实现下** 的实际能力与建议用法。

| 角色 | 建议主入口 | 可看页面 | 核心业务操作 |
|------|------------|----------|----------------|
| **管理员** | `portal.html` + `admin.html` + `index.html` | P1～P10 均可打开入口 | 全项目、全用户视角；配置成员与封面；看 AI 留痕；材料/工单/看板等与授权项目联动 |
| **施工经理** | `portal.html` / `manager.html` / `index.html` | 除 P7、P9 管理专属外均可 | 授权项目内的任务、留言、材料、工单状态、看板汇总 |
| **工人** | `worker.html` 或 `index.html` | 门户含工人入口；任务与影像 | 任务更新、留言（工作台）、现场影像只读（工人页） |
| **客户** | `client.html`、`portal.html` | P8 为主；可看工单等材料页（若授权） | 看进度/影像/节点、留言、提交工单 |
| **总包** | `portal.html` / `client.html` | 与客户类似，依授权项目 | 协同与观测 |
| **售后** | `tickets.html`、`portal.html` | 工单页 + 授权项目数据 | 处理与回访工单状态 |

**说明**：除 **P7、P9 强制管理员** 外，其它页面多数「能打开」，但 **列表数据是否为空** 取决于是否在 `project_members` 中被授权。

## 八、当前实现的局限（与方案对照）

| 能力 | 方案描述 | 当前实现 |
|------|----------|----------|
| 微信小程序 | 与 H5 同源 | **未含小程序工程** |
| 专属 APK | 白标安装包 | **未含 App 工程** |
| 实时推送 | WSS/消息推送 | 仅 **SSE 心跳占位**，无业务事件推送 |
| 多模态识图/图纸比对 | 谷歌多模态等 | 仅有 **文生文 / 文生图** 与日志留痕，无专用识图工作流页 |
| TOC/材料/售后深度 | 独立模块 | **已有** `materials.html`、`tickets.html` + 对应 API；深度 BPM 仍可后续扩展 |
| 操作手册/验收报告 | 移交文档 | **未生成 Word/PDF**，仅有 `handover.html` 与方案正文 |
